# 1주차 팀원 가이드 (데이터 수집/정제)

## 당신이 만들 것
네이버 쇼핑, iHerb에서 **제품 스펙(성분표)** + **리뷰**를 수집하는 프로그램

---

## Claude Code에 이렇게 요청하세요

### 1단계: 프로젝트 시작
```
data_manager라는 폴더를 만들고,
네이버 쇼핑과 iHerb에서 건기식 정보를 수집하는 프로그램을 만들어줘.

최종적으로 이 함수가 필요해:
collect_product_data(url, max_reviews=50)

이 함수는 URL을 받아서 "제품 스펙 정보"와 "리뷰 리스트" 두 가지를 함께 반환해야 해.
```

### 2단계: 반환 형식 지정
```
collect_product_data 함수는 반드시 이런 형태의 딕셔너리를 반환해야 해:

{
    'product_spec': {
        'product_name': 제품명 (문자열),
        'brand': 브랜드명 (문자열),
        'price': 가격 (숫자, 원 단위),
        'serving_size': 1회 섭취량 (문자열, 예: "1캡슐 (500mg)"),
        'servings_per_container': 총 섭취 횟수 (숫자),

        'ingredients': [
            {
                'name': 성분명 (문자열),
                'amount': 함량 (문자열),
                'daily_value': 일일 권장량 대비 % (문자열, 없으면 빈 문자열)
            }
        ],

        'other_ingredients': 기타 원료 리스트 (문자열 리스트),
        'warnings': 제품 주의사항 리스트 (문자열 리스트),
        'product_url': 제품 URL (문자열)
    },

    'reviews': [
        {
            'text': 리뷰 본문 (문자열),
            'rating': 평점 (1~5 숫자),
            'date': 날짜 ('2024-01-15' 형식 문자열),
            'reorder': 재구매 여부 (True 또는 False),
            'one_month_use': 한달사용 여부 (True 또는 False)
        }
    ]
}

중요한 규칙:
- 리뷰가 없으면 reviews는 빈 리스트 []
- 성분 정보가 없으면 ingredients는 빈 리스트 []
- 절대 None을 반환하면 안돼
- 문자열은 비어있으면 빈 문자열 ""로
```

### 3단계: 테스트 요청
```
collect_product_data 함수가 잘 작동하는지 테스트해줘.
네이버 쇼핑 URL 하나로 테스트해보고:

1. product_spec 안에 product_name, brand, ingredients가 있는지
2. ingredients 리스트에 각 성분의 name, amount가 있는지
3. reviews 리스트에 각 리뷰의 text, rating, date가 있는지

결과 보여줘.
```
